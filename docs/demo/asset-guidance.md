# Demo asset guidance

## Current state

No screenshot or video assets are committed in this repository. Add only captures from the actual application and label them accurately.

## Recommended captures

1. Landing view and product positioning.
2. Paste/upload workspace and indexing state.
3. Results dashboard with role comparison.
4. Score explanation drawer and evidence labels.
5. Grounded chat response with citations.
6. Tailored CV output.

## Placement and naming

Store approved assets under `docs/demo/assets/` using descriptive names, for example `upload-workspace.png` or `grounded-chat.mp4`. Add alt text and a caption describing the observed workflow, not an unverified outcome.

## Recording guidance

- Use non-sensitive sample content or material you are permitted to show.
- Keep a walkthrough focused on the actual workflow: upload, index, analyse, inspect evidence, and ask a grounded question.
- Do not overlay unsupported performance, security, or adoption claims.
- Update assets when the corresponding UI or workflow changes.
